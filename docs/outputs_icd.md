# SENTRY Outputs ICD

**Codename:** SENTRY · **Version:** 0.5.0-darkspace-integrated
**Type designation:** AN/GSQ-100(V)1 · SENTRY Node Mk I
**Interface control** for all machine-readable outputs

---

## Output catalogue

| ID | Schema file | Producer | Consumer |
|----|-------------|----------|----------|
| OUT-01 | `sensor_event.v1.json` | `LiveSensorArray` / simulation | Guardian fusion |
| OUT-02 | `alert_frame.v1.json` | `SentryGuardian` | Operator, audit, mesh |
| OUT-03 | `audit_entry.v1.json` | `AuditLogger` | Forensics, compliance |
| OUT-04 | `omen_alert.v1.json` | `MeshtasticHandler` | Mesh peers, OMEN relay |
| OUT-05 | `power_metrics.v1` (inline) | `PowerMetrics` | Power log, audit |
| OUT-06 | `deployment_manifest.v1.json` | Site planning | Install team |
| OUT-07 | `defense-readiness-sentry.json` | `run_complete_audit.py` | CI / HQ review |

---

## OUT-01: sensor_event.v1

**Rate:** 0.5 Hz live (configurable)
**Example:** [`examples/sensor_event.v1.example.json`](examples/sensor_event.v1.example.json)

| Field | Type | Description |
|-------|------|-------------|
| `channels.pir` | 0–1 | Motion proxy |
| `channels.acoustic` | 0–1 | 100–500 Hz band energy ratio |
| `channels.rf` | 0–1 | RF anomaly score |
| `channels.visual` | 0–1 | Motion score (optional camera) |
| `flags.rf_burst_2g4` | bool | Narrow burst in 2.4 GHz sweep |
| `flags.acoustic_propeller_peak` | bool | Peak in propeller band |
| `flags.jamming_suspected` | bool | Wideband RF interference |
| `flags.tamper_detected` | bool | Enclosure opened |
| `flags.fallback_channels` | object | Which channels used simulation |

---

## OUT-02: alert_frame.v1

**Example:** [`examples/alert_frame.v1.example.json`](examples/alert_frame.v1.example.json)

| Field | Description |
|-------|-------------|
| `level` | CLEAR / YELLOW / ORANGE / RED / HOLD |
| `threat_score` | 0–1 fused score |
| `rationale` | Human-readable explain string |
| `guardian_state` | FSM state |
| `sensor_event` | Embedded OUT-01 snapshot |

---

## OUT-03: audit_entry.v1

Hash chain: each entry links to previous via HMAC-SHA256.

| Event types | `boot`, `alert`, `tamper`, `tamper_wipe` |
| Secret env | `SENTRY_AUDIT_HMAC_KEY` |

---

## OUT-04: omen_alert.v1 (mesh)

**Max size:** 200 bytes (LoRa constraint)
**Example:** [`examples/omen_alert.v1.example.json`](examples/omen_alert.v1.example.json)

Emitted on ORANGE, RED, HOLD only.

---

## OUT-05: power_metrics.v1

**Example:** [`examples/power_sample.v1.json`](examples/power_sample.v1.example.json)

| Field | Unit | Source |
|-------|------|--------|
| `estimated_watts` | W | Datasheet model + duty |
| `cpu_percent` | % | psutil |
| `ram_used_mb` | MB | psutil (512 MB total on Zero 2 W) |
| `cpu_temp_c` | °C | vcgencmd (Pi only) |
| `duty_active` | bool | Guardian duty window |

---

## OUT-06: deployment_manifest.v1

Site-level plan: node positions, mesh IDs, power budget.
**Example site:** [`configs/deployment_site_alpha.json`](../configs/deployment_site_alpha.json)

---

## OUT-07: defense-readiness-sentry.json

Generated by full audit. Includes scenario peaks, probe report, failure modes, live smoke, power summary.

```
  sentry_live_audit.jsonl       # OUT-03 stream
  sentry_power.jsonl            # OUT-05 stream
  defense-readiness-sentry.json # OUT-07 snapshot

/var/lib/sentry/
  meshtastic_spool.jsonl        # OUT-04 fallback TX queue
  meshtastic_inbox.jsonl        # OUT-04 RX placeholder
```
